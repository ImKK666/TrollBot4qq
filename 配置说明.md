# TrollBot 配置说明文档

本文档详细说明了 TrollBot 的所有配置选项，帮助你根据需求定制机器人行为。

## 📁 配置文件结构

主要配置文件为 `bot_config.py`，包含以下几个配置块：
- OneBot 协议配置
- 机器人核心配置
- 认知轨道配置
- 数据库配置
- AI 服务配置
- 功能参数配置

## 🔧 详细配置说明

### OneBot 配置

```python
# OneBot WebSocket 连接地址
ONEBOT_WS_URL = os.getenv("ONEBOT_WS_URL", "ws://127.0.0.1:3001")

# OneBot HTTP API 地址
ONEBOT_HTTP_URL = os.getenv("ONEBOT_HTTP_URL", "http://127.0.0.1:3000")

# OneBot 访问令牌（可选）
ONEBOT_ACCESS_TOKEN = os.getenv("ONEBOT_ACCESS_TOKEN", "")
```

**说明**：
- `ONEBOT_WS_URL`：用于接收 QQ 消息事件的 WebSocket 地址
- `ONEBOT_HTTP_URL`：用于发送 QQ 消息的 HTTP API 地址
- `ONEBOT_ACCESS_TOKEN`：如果 OneBot 客户端配置了访问令牌，在此填入

**注意事项**：
- 确保端口与 go-cqhttp 配置一致
- WebSocket 和 HTTP 服务可以使用相同端口，但路径不同

### 机器人核心配置

```python
# 监听的群组列表
TARGET_GROUP_IDS = [761661430, 730468628]

# 重点关注的用户列表（可用于特殊处理）
CRITICAL_TARGET_LIST = [2948328662]

# 机器人自身的 QQ 号
BOT_QQ_ID = 3408830694

# 机器人对外显示的昵称
BOT_NICKNAME = "TrollBot"
```

**配置建议**：
- `TARGET_GROUP_IDS`：添加需要机器人参与的群组号
- `CRITICAL_TARGET_LIST`：添加需要特别关注的用户QQ号
- `BOT_QQ_ID`：必须与实际的机器人QQ号一致
- `BOT_NICKNAME`：建议使用简短、易识别的名称

### 认知轨道配置

```python
# 消息积累缓冲区大小
DIGESTION_BUFFER_SIZE = 50

# 消息消化间隔（秒）
DIGESTION_INTERVAL_SECONDS = 900  # 15分钟
```

**功能说明**：
- 机器人会收集群聊消息，当达到缓冲区大小或时间间隔时，进行深度分析
- 分析结果用于更新用户画像和生成话题总结

**调优建议**：
- 活跃群组：减少 `DIGESTION_BUFFER_SIZE` 到 30-40
- 安静群组：增加 `DIGESTION_INTERVAL_SECONDS` 到 1800（30分钟）
- 性能有限：增加两个数值以减少计算频率

### 数据库配置

```python
# 用户档案数据库路径
PROFILE_DB_PATH = "data/profiles.db"

# 向量记忆数据库路径
MEMORY_DB_PATH = "data/memory_db"
```

**存储说明**：
- `PROFILE_DB_PATH`：SQLite 数据库，存储用户基本信息和社交关系
- `MEMORY_DB_PATH`：ChromaDB 向量数据库，存储对话内容的语义向量

**备份建议**：
- 定期备份整个 `data/` 目录
- SQLite 文件可直接复制备份
- ChromaDB 需要复制整个目录

### AI 服务配置

#### 主要 OpenAI API 配置
```python
# 主要API密钥（用于对话生成）
OPENAI_API_KEY = "sk-your-api-key-here"
OPENAI_API_URL = "https://api.shubiaobiao.com/v1"
```

#### 嵌入向量专用配置
```python
# 嵌入向量API密钥
EMBEDDING_API_KEY = "sk-your-embedding-key-here"
EMBEDDING_API_URL = "https://fastapi.aabao.top/v1"
EMBEDDING_MODEL_NAME = "text-embedding-3-small"
EMBEDDING_API_TIMEOUT = 30
```

**多服务商支持**：
- 可以使用不同的服务商分别提供对话和嵌入服务
- 这样可以优化成本和性能
- 如果只有一个API，可以将两个配置设为相同


### LLM 模型配置

```python
# 主力对话模型
LLM_MODEL_NAME = "deepseek-r1"

# 快速回复模型
FAST_LLM_MODEL_NAME = "gemini-2.5-flash"
```

**模型选择策略**：
- `LLM_MODEL_NAME`：用于深度分析、用户画像、话题总结
- `FAST_LLM_MODEL_NAME`：用于即时回复，要求响应快

**推荐组合**：
1. **平衡型**：GPT-4 + GPT-3.5-turbo
2. **经济型**：Claude-3-haiku + Claude-3-haiku
3. **性能型**：GPT-4-turbo + GPT-3.5-turbo

### 核心逻辑配置

```python
# 快速回复参考的历史消息数量
RECENT_MESSAGES_BUFFER_SIZE = 20

# 快速回复冷却时间（秒）
QUICK_REPLY_COOLDOWN_SECONDS = 10
```

**行为控制**：
- `RECENT_MESSAGES_BUFFER_SIZE`：影响回复的上下文理解能力
- `QUICK_REPLY_COOLDOWN_SECONDS`：避免机器人过于频繁回复

**场景调优**：
- **高活跃度群**：减少缓冲区大小，增加冷却时间
- **低活跃度群**：增加缓冲区大小，减少冷却时间
- **测试环境**：设置较小的值以便观察效果

### 主动攻击配置

```python
# 主动检查间隔（秒）
PROACTIVE_ATTACK_INTERVAL_SECONDS = 300  # 5分钟

# 沉默超时阈值（小时）
PROACTIVE_ATTACK_INACTIVITY_HOURS = 2

# 活跃时间范围（小时，24小时制）
PROACTIVE_ATTACK_ACTIVE_TIME_RANGE = (9, 23)  # 9:00-23:00
```

**功能说明**：
- 当群聊沉默超过指定时间时，机器人会主动发起话题
- 只在指定时间段内执行，避免深夜打扰

**使用建议**：
- 根据群组特性调整沉默阈值
- 工作群：设置为工作时间 `(9, 18)`
- 娱乐群：可延长到 `(10, 24)`
- 如不需要此功能，设置很大的间隔值

### 记忆搜索配置

```python
# 向量搜索初步获取数量
MEMORY_SEARCH_TOP_K = 10

# 相关性距离阈值
MEMORY_SEARCH_MAX_DISTANCE = 1.2

# 最终提供给LLM的记忆数量
MEMORY_SEARCH_FINAL_MAX_COUNT = 3
```

**检索策略**：
1. 首先获取 `TOP_K` 个最相似的记忆
2. 过滤距离大于 `MAX_DISTANCE` 的记忆
3. 最多保留 `FINAL_MAX_COUNT` 个记忆供LLM参考

**性能调优**：
- 增加 `TOP_K` 可提高召回率，但增加计算量
- 减少 `MAX_DISTANCE` 可提高精确率，但可能遗漏相关信息
- `FINAL_MAX_COUNT` 影响LLM的上下文长度

## 🔒 安全配置

### 环境变量配置
建议将敏感信息通过环境变量配置：

```bash
# .env 文件
OPENAI_API_KEY=sk-your-actual-key
EMBEDDING_API_KEY=sk-your-embedding-key
ONEBOT_ACCESS_TOKEN=your-token-if-needed
```

### 配置验证
在 `bot_config.py` 中添加配置验证：

```python
import sys

# 必需配置检查
required_configs = ['OPENAI_API_KEY', 'BOT_QQ_ID']
for config in required_configs:
    if not globals().get(config):
        print(f"错误：必需配置 {config} 未设置")
        sys.exit(1)
```

## 🔄 配置热更新

对于某些配置，可以实现热更新而无需重启：

```python
# 在运行时修改配置的API端点
@app.put("/api/v1/config")
async def update_config(config_updates: dict):
    global QUICK_REPLY_COOLDOWN_SECONDS, DIGESTION_BUFFER_SIZE
    
    if 'quick_reply_cooldown' in config_updates:
        QUICK_REPLY_COOLDOWN_SECONDS = config_updates['quick_reply_cooldown']
    
    if 'digestion_buffer_size' in config_updates:
        DIGESTION_BUFFER_SIZE = config_updates['digestion_buffer_size']
    
    return {"message": "配置已更新"}
```
